# 当前刷题编辑联动设计

## 目标

在“管理”页面直接编辑刷题页面当前正在查看的题目；只有用户点击“新增题目”后才进入空白的新建状态。

## 方案选择

采用应用顶层共享当前题目 ID 的方案。刷题页在切题时上报 ID，应用顶层把 ID 传给管理页。管理页依据这个 ID 初始化和更新编辑表单。

未采用直接读取浏览器本地状态的方案，因为它不能可靠表达当前页面正在看的题目；也未引入全局状态库，因为此功能只需一个共享 ID。

## 交互

- 用户在刷题页切换题目后，再打开“管理”，管理表单显示该题目的章节、题干和答案，标题为“编辑题目”。
- 管理页处于编辑状态时，保存会更新该题目。
- “清空”按钮改为“新增题目”。点击后清空题干和答案，取消现有题目 ID，标题变为“新增题目”。章节默认保留当前题所属章节。
- 在新增状态保存成功后，表单继续保持新增状态，章节保留当前选择。
- 用户在管理页的题目列表点击任一题目时，进入该题目的编辑状态。

## 边界与错误处理

- 当前刷题题目被删除或题库尚未加载时，管理页维持既有的新增题目默认逻辑。
- 不改变题目持久化接口和 Supabase 数据结构；保存仍通过既有 `onSaveQuestion` 完成。

## 验收

- 当前刷题为第 5 题时，切到管理页能显示第 5 题全部内容。
- 点击“新增题目”后显示空白表单和“新增题目”标题。
- 编辑保存与新增保存均调用既有保存接口，且不回退成错误题目。
