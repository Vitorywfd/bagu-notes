# 拖拽排序设计

## 目标

让用户在管理页面直接拖动章节和题目到任意目标位置。新增章节和题目仍排在各自列表最后，拖放后跨设备同步。

## 交互

- 章节列表的每一行左侧显示拖动手柄。按住手柄可拖到任意章节之间后松开。
- 当前章节题目列表的每一行也显示拖动手柄。题目只能在当前章节内重新排序。
- 使用同时支持鼠标与手机触摸的 `dnd-kit`；拖动结束后才保存，不会在拖动过程中频繁写云端。
- 现有“所属章节”下拉框仍负责把题目移动到其他章节；跨章节后的细粒度顺序可再用拖动调整。

## 数据与同步

- 章节和题目继续使用现有 `sort_order` 字段，无需修改 Supabase 表结构或 RLS。
- 章节拖放后，对受影响章节重新写入连续顺序 `1..N`；题目拖放后，对该章节题目重新写入连续顺序 `1..N`。
- 标题以数字前缀开头的章节，例如 `2.MCU 篇`、`3. 操作系统篇`，会在章节排序后自动改成当前顺序号；不含数字前缀的标题保持原样。
- 云端模式先在页面上立即更新，再并行保存。任何保存失败都会刷新题库，从云端恢复真实顺序并显示错误。
- 本地预览模式调用同样的排序接口更新内存题库。

## 验收

- 可将第 1 个章节拖到最后，页面顺序和其数字前缀都正确更新。
- 可将某题拖到同章节任意位置，刷题页与题目目录按新顺序显示。
- 刷新页面、换手机或电脑后，顺序保持一致。
- 新增章节或题目默认仍在末尾。
